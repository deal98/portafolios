# Portafolios

Este repositorio contiene un sitio web de portafolio personal y una página de demos interactivas. Está pensado para mostrar habilidades de desarrollo fullstack con tecnologías como .NET, PHP/Laravel, React, Vue, Livewire, Blazor y Flutter.

## Estructura del proyecto

- `index.html` - Página principal del portafolio con secciones de bienvenida, tech stack, proyectos y contacto.
- `index-demos.html` - Página de demos interactivas y presentaciones visuales.
- `css/styles.css` - Estilos personalizados para el portafolio.
- `js/main.js` - Lógica JavaScript para animaciones, carga dinámica de proyectos y efectos de navegación.
- `js/data.json` - Datos de proyectos utilizados en la sección de proyectos de la página principal.
- `demo1-api.html` a `demo6-vue.html` - Archivos de demos independientes relacionados con tecnologías y ejemplos de proyectos.

## Qué muestra este repositorio

- Un portafolio moderno con diseño responsivo.
- Animaciones y efectos con AOS, Typed.js y Bootstrap.
- Carga dinámica de tarjetas de proyecto desde `js/data.json`.
- Demostración de componentes interactivos (contador tipo Livewire simulado, scroll suave, cambio de navbar).
- Catálogo de demos separadas para tecnologías específicas.

## Tecnologías principales

- HTML5
- CSS3
- JavaScript
- Bootstrap 5
- AOS (animaciones)
- Typed.js (efecto de texto)
- FontAwesome

Además, el contenido describe experiencia en:

- PHP / Laravel / Livewire
- C# / .NET / Blazor
- React / Vue
- Flutter
- Docker
- APIs REST
- Bases de datos MySQL, PostgreSQL, SQL Server

## Cómo usarlo localmente

1. Abrir el proyecto en un navegador.
2. Cargar `index.html` para ver el portafolio.
3. Cargar `index-demos.html` para ver las demos interactivos.
4. Si quieres editar los datos de proyectos, modifica `js/data.json`.

## Notas

- Es un proyecto estático, no necesita servidor backend para funcionar.
- Para visualizar la carga dinámica de proyectos, algunos navegadores requieren abrir el archivo con un servidor local (por ejemplo, usando Live Server en VS Code o `python -m http.server`).
- Los botones de formulario y algunos efectos son demos de comportamiento simulado.
